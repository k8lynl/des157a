Who reviewed your code and what qualifications do they possess that makes their opinions on design valid?

I had my friend review my codeâ€”he is a software engineer at Amazon. Not only does he have an undergraduate and Masters education in Computer Science, he is also currently doing frontend work in his team.

What was discussed?

One of the things he caught within my code was using an id versus a class for one of my buttons. Because the button would only be used once and only has a unique, single purpose, he told me that it would be better if it had a id instead.

Another piece of advice he gave me was using a simpler way to navigate through the pages of the content. Originally, I had a function that would hide all the pages and show the current one, but he told me that I could just hide the current page and show the next one, as opposed to hiding all of the pages.

Lastly, he caught a mistake where I was unnecessarily calling a function twice, when it only needed to be called once at the very end.

In regards to adding a randomize button, I was struggling to get the code to work when I added my word lists to a separate js file. After many attempts to troubleshoot under limited time, he suggested that I just shorten my word lists and include them locally in the script.js. This way, the script.js. file would not be too long while keeping the code functional and working.
